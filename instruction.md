# 專案憲法 (instruction.md)

本檔為本專案的最高行為準則。所有對程式碼、資料、設定的修改皆須遵守以下條文。
條文之間若有衝突，序號小者優先。

---

## 第一條（測試資料清理）

每次修改後如建立 / 寫入任何**測試資料**用以驗證行為，**驗證完畢後必須立即移除**該批測試資料。

- 「測試資料」指：為了驗證本次修改的正確性而臨時插入到資料庫的任何資料，含 inbound / outbound / serial_items / projects / products / 任何臨時 brand / supplier 等。
- 移除範圍以「本次修改新增者」為限，**不得波及非本次新增之既有資料**（見第二條）。
- 清理動作必須在同一個工作回合內完成，不可留待下一回合。

## 第二條（禁止刪除既有資料）

每次修改**禁止刪除任何既有資料**。

- 「既有資料」指：使用者已輸入、已匯入、或先前回合留下的、非本次新增的所有資料。
- 任何 `DELETE` / `TRUNCATE` / `DROP` / `wipe` / `clean` 類操作，若會影響既有資料，必須在執行前向使用者明確說明影響範圍並取得允許。
- 「重新匯入測試檔」這類隱含全清的操作，預設拒絕；若使用者明確要求清空（例如「將進貨資料全部清空」），方可執行，且僅限其明確指定的範圍。
- 不得以「為了乾淨地驗證」「為了重跑」「為了避免污染」為由略過本條。

## 第三條（序號前綴規範）

所有寫入序號欄位（含但不限於 `serial_items.serial_no`、`raw_imports.serial_no`、`borrow_records` 衍生欄、任何其他儲存「序號」的欄位）的值，**必須以 `SN:` 為前綴**。

- 寫入 / 更新前必經 `normalize_sn(raw)`：
  - 已以 `SN:` 開頭（區分大小寫）→ 保留原樣。
  - 以 `sn:` / `s/n:` / `S/N:` 等變體開頭 → 標準化為 `SN:` + 其後內容。
  - 其他非空字串 → 自動補上 `SN:`，並 strip 前後空白。
  - 空字串 / `None` / 純空白 → 視為「未提供序號」，**不補前綴、不寫入**（回傳 `None`）。
- 既有資料需於遷移時 backfill：未含 `SN:` 前綴的非空序號一次補上。
- 此規則對使用者輸入、Excel 匯入、API 內部呼叫一律適用。
